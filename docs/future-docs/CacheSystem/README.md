# مستندات Cache System پروژه

در این پروژه، کش (Cache) نقش مهمی در بهبود سرعت، کاهش بار دیتابیس و تجربه کاربری ایفا می‌کند. در ادامه، ساختار و استراتژی‌های کش توضیح داده شده است:

---

## 1. نوع کش و تنظیمات
- **نوع کش:** Redis (با استفاده از django-redis)
- **تنظیمات:**
  - تعریف در settings.py با کلید `CACHES['default']`
  - استفاده از Redis با آدرس و رمز عبور از env
  - تنظیمات پیشرفته: pool connection، فشرده‌سازی، سریالایزر، key prefix
  - تایم‌اوت‌های مختلف برای داده‌های مختلف (دسته‌بندی، امتیاز کاربر، سبد خرید و ...)
- **کش سشن:**
  - سشن کاربران نیز روی کش Redis ذخیره می‌شود (SESSION_ENGINE = 'django.contrib.sessions.backends.cache')

---

## 2. استراتژی‌های کش
- **کش صفحه (Page Cache):**
  - استفاده از دکوریتور `@cache_page(timeout)` برای کش کل خروجی viewها (مثلاً لیست منابع، لیست نویسندگان، ...)
- **کش داده (Data Cache):**
  - استفاده از `cache.get_or_set(key, func, timeout)` برای کش کردن کوئری‌های سنگین یا داده‌های تکراری (مثلاً جزئیات کتاب، لیست دسته‌بندی‌ها، امتیاز کاربر، ...)
- **کش بر اساس کلید پویا:**
  - کلیدهای کش معمولاً شامل شناسه، اسلاگ یا شماره صفحه هستند (مثلاً refrence_detail_{slug}، authors_page_{page})
- **پاک‌سازی کش:**
  - پس از تغییر داده‌های مهم (مثلاً ثبت کامنت، ثبت سفارش، ویرایش کتاب)، کش‌های مرتبط با آن داده با توابعی مثل `clear_refrence_cache` یا `cache.delete_many` پاک می‌شوند.
- **کش کوئری‌های مرتبط:**
  - برای داده‌های مرتبط (مثلاً تبلیغات مرتبط با کتاب، لیست زیرمجموعه‌های دسته‌بندی) نیز کش جداگانه تعریف می‌شود.
- **کش context processors:**
  - داده‌های context پرکاربرد (دسته‌بندی‌ها، سطوح مهارت، سری‌ها) با کش get_or_set در context_processors.py ذخیره می‌شوند.
- **کش بر اساس وضعیت احراز هویت:**
  - دکوریتور `cache_on_auth` برای کش کردن خروجی view بر اساس وضعیت لاگین کاربر (کلید کش شامل is_authenticated)

---

## 3. مثال‌های کاربردی
- کش لیست منابع: `@cache_page(60*15)` و کلیدهای refrence_list_page_{page}
- کش جزئیات کتاب: `cache.get_or_set(f'refrence_detail_{slug}', ...)`
- کش لیست نویسندگان: `@cache_page(60*15)` و کلید authors_page_{page}
- کش امتیاز و سبد خرید کاربر: کلیدهای user_points_{user_id} و cart_count_{user_id}
- کش نتایج جستجو و فیلتر: کلیدهای پویا بر اساس پارامترهای جستجو

---

## 4. نکات مهم
- **مدت زمان کش:** معمولاً 15 دقیقه برای داده‌های لیستی و 5 دقیقه برای داده‌های حساس به تغییر
- **پاک‌سازی هوشمند:** پس از هر تغییر مهم، کش‌های مرتبط پاک می‌شوند تا داده‌های جدید نمایش داده شود
- **پیشگیری از کش بیش از حد:** داده‌های شخصی یا حساس (مثلاً اطلاعات کاربر) با کلیدهای خاص و مدت کوتاه کش می‌شوند
- **سازگاری با چند سرور:** استفاده از Redis باعث می‌شود کش بین چند سرور به اشتراک گذاشته شود

---

> **نکته:** کش مناسب باعث افزایش چشمگیر سرعت سایت و کاهش بار دیتابیس می‌شود، اما باید مراقب اعتبارسنجی و پاک‌سازی به موقع آن بود.
